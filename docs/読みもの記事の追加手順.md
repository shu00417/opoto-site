# 「読みもの（CONTENTS）」記事の追加手順

OPOTO公式サイトの **読みもの** ページ（https://opoto.jp/contents ）に、noteの連載記事を追加するための手順書です。

## この仕組みのしくみ（かんたんな説明）

- 1記事 = 1つのMarkdownファイル（`src/content/contents/` に置く）で管理します。
- サムネイル画像は、**noteの記事から写真とタイトルを自動で取ってきて生成**します（手作業でスクショを作る必要はありません）。
- カードをクリックすると、その **note記事が別タブで開きます**（サイト内に記事本文は作りません）。
- シリーズは2つ：**鳥取しごとびと**（値: `sigotobito`）と **あの人のフォトライフ**（値: `photolife`）。

---

## 記事を1本追加する手順

### 手順1. noteに記事を公開する
先にnoteで記事を公開し、その **記事URL** を用意します。
例: `https://note.com/opoto_tottori/n/xxxxxxxxxxxx`

### 手順2. サムネイル画像を生成する
ターミナルで `opoto-site` フォルダに移動し、次のコマンドを実行します。

```bash
npm run gen-thumb "<note記事URL>" "public/CONTENTS/<シリーズ>/<連番>.png"
```

- `<シリーズ>` … `sigotobito` または `photolife`
- `<連番>` … 続き番号（例: 鳥取しごとびとの4本目なら `004`）

例（鳥取しごとびとの4本目）:
```bash
npm run gen-thumb "https://note.com/opoto_tottori/n/xxxxxxxx" "public/CONTENTS/sigotobito/004.png"
```

> これで、写真の下にタイトルが入ったサムネ（1200×1044px）が自動で作られます。
> タイトル末尾の「｜Photo and Culture OPOTO」は自動で消えます。

### 手順3. 記事のMarkdownファイルを作る
`src/content/contents/` に、次の内容で新しいファイルを作ります。
ファイル名は **`公開日-英語の名前.md`**（例: `2026-07-25-example.md`）。

```markdown
---
title: "（noteのタイトル。カード下には出ないが管理用に入れておく）"
series: "sigotobito"
date: 2026-07-25
thumbnail: "/CONTENTS/sigotobito/004.png"
note: "https://note.com/opoto_tottori/n/xxxxxxxx"
draft: false
---
```

- **date** … 記事の並び順に使われます（**新しい日付が左に**並びます）。noteの公開日を入れておくと自然です。
- **thumbnail** … 手順2で作った画像のパス（先頭は `/CONTENTS/...`）。
- **note** … カードのリンク先。空にするとnoteのトップに飛びます。
- **draft** … `true` にすると公開されません（下書き）。

> ※ コードを触らず **管理画面**（https://opoto.jp/admin/ ）の「読みもの (CONTENTS)」からも追加できます。
> その場合もサムネ画像は先に手順2で作り、管理画面の「サムネイル画像」でアップロードしてください。

### 手順4. 本番に反映する
変更を保存して公開します。以下のどちらかで行えます。

- **Claudeに頼む場合**: 「読みものに記事を追加したので本番反映して」と伝える。
- **自分でやる場合**:
  ```bash
  git add -A
  git commit -m "Add 読みもの article"
  git push
  ```
  プッシュするとNetlifyが自動でデプロイし、1〜2分で https://opoto.jp/contents に反映されます。

## 「あの人のフォトライフ」を始めるとき

まだ記事が0件のシリーズです。1本目を追加するときは：

1. 画像用フォルダを作る（初回のみ）: `public/CONTENTS/photolife/`
2. あとは上の手順と同じ（`<シリーズ>` を `photolife` にする）

---

## 補足・困ったとき

- **サムネの見た目**: 全カードを同じ高さ（1200×1044px）に統一しています。タイトルが短い記事は、余白の中でタイトルが上下中央に配置されます。もっと長いタイトルが来ても自動で高さが広がり、文字が切れません。
- **フォント**: サイトの見出しと同じ明朝（Zen Old Mincho）。`scripts/fonts/ZenOldMincho-Bold.ttf` を使用。
- **画像が更新されない**: ブラウザのキャッシュが原因のことがあります。強めの再読み込み（Mac: `Cmd + Shift + R`）を試してください。
- **生成の細かい設定**（余白・文字サイズなど）は `scripts/gen-thumb.mjs` の上部にまとまっています。
